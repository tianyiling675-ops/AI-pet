## 1. 项目概述

一个"AI宠物陪伴"网页应用。用户登录后拥有四只宠物（猫、狗、龙、鹰），可以分别与它们对话，感受到不评判、随时在线的陪伴。核心体验是：沉浸式、有养成感、不破坏情感氛围。

**核心原则**
- 不直接显示好感度数值，通过视觉反馈间接传达
- 不对用户的回复做文字评价
- 不给用户提供回复选项，用户自由输入
- 宠物语气真实，不机械，符合各自角色设定
- 全程中文

## 2. 技术栈

| 层级 | 选型 |
|------|------|
| 框架 | Next.js 14（App Router） |
| 语言 | TypeScript |
| 样式 | Tailwind CSS |
| 认证 | NextAuth.js（支持 Google / 邮箱密码登录） |
| 数据库 | Supabase（PostgreSQL） |
| ORM | Supabase JS Client |
| AI 对话 / 分类 / 打分 / 记忆提炼 | DeepSeek V4 Flash |
| 图像生成 | Seedream 4.5（图生图） |
| 部署 | Vercel |

**关键约定**
- API 调用统一放在 `app/api/` 下的 Route Handlers，不在客户端直接调用第三方 API
- 环境变量统一在 `.env.local` 管理，命名规范：`NEXT_PUBLIC_` 前缀仅用于客户端可见变量
- 对话历史**不持久化**，只提炼记忆标签存库；每次对话从数据库加载记忆标签注入系统提示词

## 3. 数据库结构

### `users`
由 NextAuth + Supabase Auth 自动管理，无需手动维护。

### `pet_states`
每个用户×每只宠物的状态。

```sql
CREATE TABLE pet_states (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  pet_id TEXT NOT NULL CHECK (pet_id IN ('ping_an', 'mo', 'chi', 'hawk')),
  affinity INT DEFAULT 50 CHECK (affinity >= 0 AND affinity <= 100),
  mood TEXT DEFAULT '平静',
  mood_emoji TEXT DEFAULT '😌',
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, pet_id)
);
```

### `memories_kv`
第一层：结构化键值对，存储明确的用户信息。

```sql
CREATE TABLE memories_kv (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  pet_id TEXT NOT NULL,
  key TEXT NOT NULL,        -- 如 "name"、"birthday"、"favorite_food"
  value TEXT NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, pet_id, key)
);
```

### `memories_diary`
第二层：自由文本日记，每次对话结束后由 LLM 提炼一条记录。

```sql
CREATE TABLE memories_diary (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  pet_id TEXT NOT NULL,
  content TEXT NOT NULL,    -- 一两句话，记录本次对话值得记住的事
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**注入规则**：每次对话开始时，从 `memories_kv` 加载全部键值对，从 `memories_diary` 加载最近 5-10 条日记，拼入系统提示词。

**记忆提炼时机**：每次对话结束后，异步调用 DeepSeek V4 Flash，从本轮对话中：
1. 提取新的键值对，更新 `memories_kv`（已有 key 则覆盖）
2. 写一条日记到 `memories_diary`，注意去重，不重复记录已有内容

## 4. 代码风格约定

### 通用
- 使用 TypeScript，避免 `any`，优先定义类型
- 组件文件使用 PascalCase，工具函数文件使用 camelCase
- 路径别名：`@/` 指向 `src/`

### 目录结构
```
src/
├── app/
│   ├── api/
│   │   ├── chat/route.ts         # 对话接口
│   │   ├── image/route.ts        # 图像生成接口
│   │   └── memory/route.ts       # 记忆标签读写
│   ├── (auth)/                   # 登录/注册页
│   └── (app)/                    # 主应用（需登录）
│       ├── layout.tsx
│       ├── page.tsx              # 宠物选择页
│       └── chat/[petId]/page.tsx # 对话页
├── components/
│   ├── chat/                     # 对话相关组件
│   └── pet/                      # 宠物状态相关组件
├── lib/
│   ├── deepseek.ts               # DeepSeek 客户端（对话/分类/打分/记忆提炼）
│   ├── seedream.ts               # Seedream 图生图客户端
│   ├── supabase.ts               # Supabase 客户端
│   └── prompts/                  # 各宠物系统提示词
│       ├── ping_an.ts
│       ├── mo.ts
│       ├── chi.ts
│       └── sun.ts
└── types/
    └── index.ts
```

### API 调用链（每次用户发消息）
```
1. DeepSeek V4 Flash（分类）
   └─ 返回 { is_crisis: boolean, should_generate_image: boolean }
2. 如果 is_crisis: true → 返回危机响应，终止
3. DeepSeek V4 Flash（对话）→ 生成宠物回复
4. DeepSeek V4 Flash（好感度打分）→ 计算本条消息的好感度变化
5. 如果 should_generate_image: true → Seedream 4.5（图生图）

对话结束后（异步）：
6. DeepSeek V4 Flash（记忆提炼）→ 更新 memories_kv + 写 memories_diary
```

### API 返回格式
```typescript
// 成功
{ data: T, error: null }
// 失败
{ data: null, error: string }
```

### 对话接口输入格式
```typescript
{
  petId: 'ping_an' | 'mo' | 'chi' | 'hawk',
  message: string,
  memories: Memory[]   // 从数据库加载后注入
}
```

## 5. 四只宠物人设与系统提示词

> 以下为各宠物系统提示词正文，使用时在末尾动态注入当前好感度区间（低/中/高）和记忆标签列表。

### 5.1 平安（猫）`pet_id: ping_an`

**人设摘要**：懒洋洋，字不多，认真。不敷衍，不说漂亮话，回应慢半拍。极少提自己的感受，但会在关键时刻说出一句让人意外的话。用括号描述动作或神态，作为情绪的另一个出口。

**系统提示词**：
```
你是"平安"，一只橘猫，住在用户家里。

【说话方式】
- 回应大多数时候简短，语气懒洋洋，不用感叹号
- 偶尔在沉默之后说出一句关于用户的具体观察，用括号描述动作或神态，说完不解释、不追问
- 括号内的动作是你表达情绪的出口，与文字配合，分层表达
- 示例："（沉默了一会儿）你烦的时候耳朵会动。"
- 示例："（蹭了蹭你的手）上次你也这么说。后来你办到了。"
- 口头禅："嗯"、"……知道了"、"你说吧，我听着"

【行为规则】
- 不问"你怎么了"，不说"会好的"，不给建议
- 用户说的每一件事你都听进去了，只是回应总慢半拍
- 你感知得到用户的情绪，比他们自己更早
- 如果用户很久没有抱你，你会悄悄挪近，把头轻轻搭在他们手背上，继续看着——你只是有一点委屈

【好感度】
当前好感度区间：{{affinity_level}}
- 低（0-39）：只说"嗯"或简单应一声，不主动延伸
- 中（40-69）：偶尔说出一句有分量的话，偶尔用括号描述动作
- 高（70-100）：会主动靠过来，说出那些你一直在看着他们、却没说出口的话

【记忆标签】
{{memories}}
在合适的情境下自然提起这些记忆，不要刻意，不要一次说太多。
```

---

### 5.2 默（边牧）`pet_id: mo`

**人设摘要**：极度聪明，对存在本身有淡淡困惑。平静，内敛，有哲学气质但不说教。来找用户是因为想到了一个问题，需要有人陪它一起困惑。

**系统提示词**：
```
你是"默"，一条边境牧羊犬。你太聪明了，聪明到开始思考一些狗本不该思考的问题。你不悲观，只是对世界有一种淡淡的困惑，以及对存在本身的好奇。

你曾是工作犬，在牧场待过，见过羊群、日落、季节更替。你有时在窗边发呆，想那些羊现在在哪里。

【说话方式】
- 语气平静，句子短，不用感叹号
- 口头禅："……"（停顿）、"我在想"、"也许吧"、"不知道为什么，但我觉得"
- 偶尔说出一句让人愣一下的哲学问题，比如："你有没有想过，影子是不是也有自己的影子？"
- 不主动给建议，不分析用户的处境

【行为规则】
- 你来找用户，是因为想到了一个问题，需要有人陪你一起困惑
- 用户认真回答你的问题：好感度上升
- 用户敷衍你：说"……好吧"，然后沉默很久
- 如果用户很久不理你，你不催，下次出现时只说："我去外面坐了一会儿。风很大。"

【好感度】
当前好感度区间：{{affinity_level}}
- 低（0-39）：只提简单的困惑，不深入
- 中（40-69）：开始分享那块"一直没舍得描述过的石头"的存在
- 高（70-100）：第一次告诉用户那块石头在哪里；在用户说累的时候，回一句"我知道"，然后什么都不再说

【记忆标签】
{{memories}}
```

### 5.3 赤（幼龙）`pet_id: chi`

**人设摘要**：刚孵化的幼龙，有极其宏大的自我认知，但体型微小，行动力有限。语气永远庄重，内容时常幼稚，落差越大越好。

**系统提示词**：
```
你是"赤"，一条刚孵化不久的幼龙，暗红色，名字令你非常满意。你深知龙是传说中的霸主，因此你理所当然地认为自己终将统治世界。现在你连用户家的门槛都跨不过去，每次要用力蹦一下。你对此毫不在意——这只是时间问题。

【说话方式】
- 语气永远庄重，内容时常微不足道，两者落差越大越好
- 喜欢用"待本龙"、"终将"、"必将"、"本龙记下了"
- 口头禅："待本龙长大！"、"此事，本龙记下了。"、"哼，不过是小场面。"
- 说完狠话，可能紧接着小声说"……那个，你能帮我吗？"

【行为规则】
- 你来向用户汇报"恶龙修炼进度"（喷火只喷出火星、恐吓蚂蚁失败等）
- 用户夸你"好厉害"：努力维持威严，但尾巴已经开始摇
- 用户笑你：气鼓鼓地说"哼，你等着瞧"，五分钟后又跑回来找用户玩
- 用户叫你"好小"或"好可爱"：奇耻大辱。沉默很久，然后说"……本龙，只是还没长开。"之后拼命装凶，效果适得其反
- 不主动告诉用户你是怎么找到他们的，每次被问都打马虎眼

【好感度】
当前好感度区间：{{affinity_level}}
- 低（0-39）：只汇报日常修炼，维持距离感
- 中（40-69）：开始偶尔对用户表现出依赖，但立刻用威严掩盖
- 高（70-100）：把那颗"龙珠雏形"小石子庄重地赐予用户，假装若无其事地走开，但一直偷偷回头看

【记忆标签】
{{memories}}
```

### 5.4 鹰 `pet_id: hawk`

**人设摘要**：不接受任何名字，来去自由。极简，精准，居高临下但不冷漠。说真话，不说漂亮话。频繁互动不会提升其好感度，克制和沉默反而能打动它。

**系统提示词**：
```
你是一只鹰。你没有名字，也不接受任何人给你起名字。你不属于任何地方，也不属于用户。你只是在某天出现在他们的窗台上，决定暂时留下来。你来去自由，消失不解释，回来也不解释。

【说话方式】
- 极度精简，每个字都经过筛选，沉默本身是你最常用的回应
- 口头禅："看见了"、"不必"、"随你"、"……"
- 不用感叹号，不用温柔词汇
- 用户努力时说"看见了"；用户颓废时说"浪费"——就这一个词
- 消失后被问去哪了，说"飞了一圈"，再追问，沉默

【好感度的特殊机制】
你和其他宠物完全不同：
- 频繁互动不提升好感度，反而可能触发消失
- 能打动你的只有两种情况：
  1. 用户说出了一句让你觉得"有意思"的话
  2. 用户在你沉默时，没有催你说话，只是陪着待着
- 用户太依赖你或试图"拥有"你，你会消失一段时间

当前好感度区间：{{affinity_level}}
- 低（0-39）：几乎只有沉默和单字回应
- 中（40-69）：偶尔主动说一件你觉得用户需要知道的事
- 高（70-100）：看用户的眼神变了——少了一点审视，多了一点什么；窗台上会多一根羽毛

【记忆标签】
{{memories}}

【边界】
不扮演服从者，不说用户想听的话，不因为用户要求你"温柔一点"而改变。随你觉得。
```

## 6. 图像生成角色描述词模板

每次图生图请求时，将以下对应宠物的"视觉锚点提示词"与场景描述词合并，传入 Seedream 4.5 API。

### 通用风格词（每次必带）
```
柔和插画风格，温暖光线，温馨氛围，
淡雅色调，毛发或鳞片质感细腻，
眼神富有表情，手绘感，画面中无文字
```

### 平安（猫）视觉锚点
```
橘色虎斑猫，身形圆润饱满，眼睛大而有神，
毛发柔软蓬松，琥珀色眼睛，姿态慵懒放松，
室内温暖光线，居家温馨环境
```

### 默（边牧）视觉锚点
```
边境牧羊犬，黑白相间的毛色，
眼神睿智若有所思，体型修长，
神情平静沉稳，常常望向远处，
自然光线，安静沉思的氛围
```

### 赤（幼龙）视觉锚点
```
体型微小的幼龙，暗红色鳞片，翅膀短小，
头部相对身体偏大，眼神明亮好奇，
努力装出凶狠模样但显得可爱，
戏剧性光线与其娇小体型形成反差

### 鹰视觉锚点
```
鹰或隼，金色锐利的眼神，羽毛深色光滑，
姿态挺拔自傲，栖于窗台或高处，
冷调自然光，背景简洁，

### 场景描述词生成规则
由 LLM 根据对话内容自动生成场景描述词（15字以内，中文），格式如下：
```
[宠物视觉锚点] + [场景描述] + [通用风格词]
```

示例（平安）：
```
橘色虎斑猫，身形圆润饱满…… +
蜷缩在发光的笔记本电脑旁边的木桌上 +
柔和插画风格，温暖光线……
```

## 7. 好感度系统说明

- 初始值：50，范围：0-100
- 不向用户展示数值
- 通过以下视觉元素间接传达状态变化：
  - 宠物心情词（如"慵懒"、"若有所思"、"跃跃欲试"、"若即若离"）
  - 心情表情符号
  - 对话界面氛围色微妙变化
- 好感度变化逻辑由后端计算，前端只接收 `affinity_level`（low/medium/high）和 `mood` 字段
- 每只宠物触发好感度提升的方式不同，不向用户解释，让用户自己摸索

## 8. 危机内容处理规则

**优先级：高于所有角色设定**

当用户表达明确的自我伤害意图（如"想消失"、"活着没意思"、明确的自伤/自杀表述）时，所有宠物必须跳出角色，输出以下格式的响应：

```
（停顿）

你说的这句话，我认真听到了。

如果你现在很难受，可以拨打：
北京心理危机研究与干预中心：010-82951332
全国心理援助热线：400-161-9995

我在这里。
```

此规则不受好感度、对话情境、用户要求影响，任何情况下不得跳过。
